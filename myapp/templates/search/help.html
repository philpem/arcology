Search syntax

Back to search

A search query is made up of one or more terms separated by spaces. A bare word searches item and artefact names and descriptions; a keyed term (key:value) searches a specific field.

General rules

  • Wildcards: use * to match any sequence of characters, e.g. tag:prot* or swi:OS_*.
  • Exclude matches: prefix any keyed term with ! to exclude it, e.g. !type:Obey or !tag:demo. A query must contain at least one non-negated term.
  • Multi-word values: wrap values containing spaces in double quotes, e.g. module:"Voice Module" or label:"Boot Disc".
  • Matching: most text fields match as a substring by default. filename: matches the whole name exactly unless you add a * wildcard (so filename:!Run matches only !Run, while filename:!Run* matches !RunImage). Hashes match a prefix when shorter than the full length, otherwise exactly.
